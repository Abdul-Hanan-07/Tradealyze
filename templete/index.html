01 — Introduction
Welcome to Tradealyze Terminal

Tradealyze Terminal is a real-time trading simulator powered by a C++ matching engine communicating over WebSockets. It replicates the interface and workflow of professional crypto exchanges, letting you practice strategies, test backend logic, and observe live market mechanics — all without real funds.

This manual covers every panel of the interface. Whether you're new to trading or setting up the backend engine, you'll find the relevant section below.

ℹ️ This is a simulator. No real cryptocurrency is traded or held. Balances and prices are generated by the connected C++ backend.
02 — Platform Overview
Navigating the Interface

The terminal is organized as a Single Page Application with three main views, accessible from the top navigation bar.

⬛
Dashboard
The primary trading screen. Shows the live order book, price chart, order entry form, and transaction history.
◈
Markets
A sortable table of all available trading pairs with live prices and 24-hour statistics. Click Trade Now to load any pair into the dashboard.
⊕
User Manual
This documentation screen. Use the sidebar to jump between sections.
💼
Wallet Bar
Live BTC, ETH, and USDT balances pushed directly from the C++ engine. Always visible in the top-right corner.
03 — Order Book
Reading the Order Book

The order book is a live list of all pending buy (bid) and sell (ask) orders in the market. It sits in the left column of the Dashboard.

These are offers from sellers. They are shown in red. The lowest ask is the cheapest price at which someone is currently willing to sell. Orders are sorted so the cheapest ask is closest to the spread.
These are offers from buyers. Shown in green. The highest bid is the best price a buyer is willing to pay. The depth bar behind each row shows its relative size compared to the total visible liquidity.
The spread is the gap between the lowest ask and the highest bid. A tighter spread means a more liquid market. A wider spread means less liquidity and potentially higher cost to trade. The spread is shown between the two sides, highlighted in amber.
The "Total" column shows the cumulative amount of the asset available from that price level inward (toward the spread). It helps you understand how much liquidity exists at each price level.
04 — Placing Orders
How to Place an Order

The order entry form is in the right column of the Dashboard. Follow these steps to place your first simulated trade.

1
Choose a side — Click Buy / Bid if you want to purchase the asset, or Sell / Ask if you want to sell it.
2
Select an order type — Limit, Market, or Stop (see the Order Types section for details).
3
Set your product — Type the trading pair you want, e.g. ETH/BTC. You can also click Trade Now in the Markets view to pre-fill this.
4
Enter price and amount — Price is denominated in the quote currency (e.g. BTC). Amount is in the base currency (e.g. ETH). The Estimated Total updates automatically.
5
Use Quick Fill (optional) — The 25% / 50% / 75% / Max buttons calculate an amount based on your available balance.
6
Submit — Click the Place Order button. The order is sent to the C++ backend via WebSocket. A confirmation appears below the button, and the trade appears in the Transaction History panel.
⚠️ If the WebSocket is not connected, orders will not be forwarded to the backend engine. Watch the connection status in the top-right corner.
05 — Order Types
Limit, Market, and Stop Orders
You specify the exact price you want to pay or receive. The order will only execute if the market reaches your price. Limit orders give you price control but are not guaranteed to fill. Best for: entering at a specific price level you've identified.
Executes immediately at the best available price in the order book. The price field is disabled — the backend will match you against the top ask (if buying) or top bid (if selling). Best for: entering or exiting a position quickly when speed matters more than price.
A stop order becomes a market order once the price crosses a trigger level you specify. It is used to limit losses (stop-loss) or lock in profit (take-profit). The price field is your trigger price. Best for: automated risk management without needing to watch the screen.
06 — Markets View
Browsing Trading Pairs

The Markets view displays every trading pair loaded from the C++ backend, with live prices, 24-hour statistics, and sparkline trend charts.

Use the filter chips (All, BTC Pairs, ETH Pairs, USDT Pairs) to narrow the list by quote currency. Use the search box to find a specific pair by name. Click any column header to sort by that metric.

Clicking Trade Now on any row switches you to the Dashboard and pre-fills the product field with that pair, so you can begin trading immediately.

ℹ️ Developers: call updateMarkets(productsArray) from your WebSocket handler to populate this table with live data from the C++ engine.
07 — Transaction History
Reviewing Your Trades

The Transaction History table at the bottom of the Dashboard records every order submitted during your session. Columns show the time, side (BUY/SELL), product, execution price, amount, total cost, and status.

The Next Timeframe button sends an advance-timeframe signal to the C++ engine, progressing the simulator from T+0 (Genesis) through morning, midday, afternoon, and after-hours sessions. Session statistics (total trades and volume) are tracked in the panel header.

08 — Backend Connection
WebSocket & C++ Integration

The terminal connects to the C++ engine at ws://localhost:9002 on load. The connection status is displayed in the top-right corner of the navigation bar.

{ type: "balances", btc, eth, usdt } — Update wallet balances
{ type: "orderbook", asks, bids } — Push a new order book snapshot
{ type: "trade", tx } — Push a completed trade to transaction history
{ type: "chart", points } — Push price series [{x, y}] for the chart
{ action: "submit_order", side, product, price, amount, orderType }
{ action: "next_timeframe" }
⚠️ Ensure the C++ server is running on port 9002 before opening the terminal. The UI will continue to run with simulated data even when disconnected, but orders will not be processed.
